 D. POIN-POIN YANG HARUS DIPRESENTASIKAN
================================================================================

1. ARSITEKTUR FRONTEND
   - Pola layout: Blade template inheritance (@extends, @section, @yield)
   - Komponen reusable: Blade components (<x-component>)
   - Pemisahan layout: public vs admin vs auth
   - Stack system: @push/@stack untuk CSS/JS per halaman

2. DESIGN SYSTEM & UI
   - Warna brand: Biru Navy (#172554 primary), Slate tones untuk teks
   - Tipografi: Inter (body), Barlow Condensed (heading)
   - Card design: rounded corners, shadow, hover effects
   - Responsive design: 3 breakpoint (mobile, tablet, desktop)

3. FITUR INTERAKTIF
   - Hero carousel dengan auto-slide dan manual indicators
   - Swiper.js slider untuk profil guru
   - Counter animasi statistik sekolah
   - Dropdown navigasi bertingkat (hover di desktop, click di mobile)
   - Running text (marquee) di topbar
   - Tanggal otomatis format Indonesia (JavaScript)
   - Google Maps embed di footer

4. RESPONSIVITAS
   - Fixed top-bar + navbar yang responsif
   - Navbar collapse menjadi hamburger menu di mobile
   - Grid system Bootstrap untuk layout konten
   - Hiding/showing elemen sesuai ukuran layar (d-none, d-md-flex)

5. FORM & INTERAKSI USER
   - Formulir kontak/pesan pengunjung di beranda
   - Validasi form (client-side & server-side via Blade @error)
   - Flash message untuk feedback (sukses/gagal)
   - File upload preview (gambar berita, foto guru, dll)

6. HALAMAN ADMIN
   - Dashboard dengan statistik ringkasan
   - Tabel data dengan aksi CRUD
   - Form input dengan validasi
   - Sidebar navigasi dengan icon SVG kustom
   - Breadcrumb untuk navigasi hierarkis

7. OPTIMASI & BEST PRACTICES
   - Lazy loading untuk Google Maps iframe
   - Preconnect untuk Google Fonts
   - SVG inline icons (tanpa HTTP request tambahan)
   - Semantic HTML (nav, main, footer, article)
   - SEO: meta description dinamis, title tag, heading hierarchy
   - Accessibility: aria-label, alt text, semantic elements

================================================================================
 F. TIPS PRESENTASI
================================================================================

1. BUKA DEMO LANGSUNG (php artisan serve → localhost:8000)
   - Tunjukkan navigasi halaman publik dari beranda hingga kontak
   - Tunjukkan responsivitas dengan resize browser
   - Tunjukkan hero carousel berpindah otomatis

2. TUNJUKKAN KODE PENTING
   - Buka layouts/public.blade.php → jelaskan struktur layout
   - Buka welcome.blade.php → jelaskan hero carousel & sections
   - Buka components/admin-icon.blade.php → jelaskan komponen reusable
   - Buka public/css/admin-panel.css → tunjukkan custom styling admin

3. JELASKAN KEPUTUSAN DESAIN
   - Kenapa Bootstrap 5 untuk publik, Bootstrap 4 untuk admin?
     → Admin panel menggunakan template AdminLTE berbasis Bootstrap 4
   - Kenapa CSS inline di layout, bukan file terpisah?
     → Untuk mengurangi HTTP request di halaman publik
   - Kenapa SVG inline bukan icon library?
     → Performa: tidak perlu download font file tambahan

4. SIAPKAN JAWABAN UNTUK PERTANYAAN UMUM
   - "Bagaimana data dari database bisa tampil di halaman?"
     → Controller query data → kirim ke view → Blade render {{ $variable }}
   - "Bagaimana halaman bisa responsive?"
     → Bootstrap grid + media queries custom
   - "Apa bedanya @extends vs @component?"
     → @extends untuk layout inheritance, @component untuk reusable parts
   - "Kenapa tidak pakai React/Vue?"
     → Proyek ini server-rendered, Blade sudah cukup untuk kebutuhan

5. HIGHLIGHT KONTRIBUSI UNIK
   - 12 halaman publik + 1 landing page besar (59KB)
   - 72KB custom CSS untuk admin panel
   - 5 Blade components reusable
   - Design system konsisten (warna, tipografi, spacing)
   - Integrasi Google Maps, Swiper.js, format tanggal Indonesia
